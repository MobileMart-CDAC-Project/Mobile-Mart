---

### What Happens After You Report

1. You will receive an acknowledgment within **48 hours**.
2. The vulnerability will be reviewed and validated.
3. Severity will be assessed.
4. A fix will be developed and tested.
5. A patched version will be released for supported versions.

If the vulnerability is accepted, you may be credited (if desired).

If the report is declined, a clear explanation will be provided.

---

## Responsible Disclosure Guidelines

We ask that researchers:

- Do not publicly disclose the issue before a fix is released.
- Avoid accessing, modifying, or deleting user data.
- Limit testing to proof-of-concept validation only.
- Provide detailed reproduction steps where possible.

---

## Security Best Practices

This project follows industry security practices including:

- Environment-based configuration for secrets
- JWT authentication with secure secret management
- Input validation and sanitization
- Regular dependency updates
- Principle of least privilege
